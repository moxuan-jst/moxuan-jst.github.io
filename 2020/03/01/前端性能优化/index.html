前端性能优化

本篇博文共计 697 字   |   阅读约 2 分钟
发表于 2020-03-01 更新于 2022-08-13 分类于 前端 ， 笔记 阅读次数：
  1. URL 层面到显示页面，涉及到网络层面，有三个重要过程
  • DNS 解析
  • TCP 连接
  • HTTP 请求/响应（优化的核心）
    • 减少请求次数
    • 减少单次请求所花费的时间
  1. 浏览器缓存策略

  • 浏览器缓存机制有四个方面：

    • Memory Cache（内存）

      • 指存在于内存中的缓存，是浏览器最先尝试命中的一种缓存，也是响应速度最快的响应。
      • Base64 格式的图片以及较小的 JS，CSS 文件会使用。

    • service Worker cache

      • 独立于主线程之外的 JS 线程，脱离于浏览器窗体，无法直接访问 DOM。
      • 可实现离线缓存，消息推送和网络代理等。

    • HTTP cache

      • 分为强缓存和协商缓存

        • 优先级较高的是强缓存，在命中强缓存失败的情况下，才会走协商缓存

      • 强缓存是利用 http 头中的 Expires和cache-control两个字段来控制的

        • cache-control相对于Expires更加准确，优先级也更高，同时出现时以cache-control为准。

      • 协商缓存依赖于服务端与浏览器之间的通信。

      • HTTP 缓存策略

      • public 和 private

        • public：既可以被浏览器缓存，也可以被代理服务器缓存。
        • private：只能被浏览器缓存（默认）。

      • no-store 和 no-cache

        • no-store：不适用任何缓存。
        • no-cache：直接向服务端确认该资源是否过期。

    • push cache

      • 只有当前三种均未命中时激活。
      • 是一种存在于会话阶段的缓存，当 session 终止时，缓存也随之释放。
      • 不同页面只要共享了同一个 HTTP2 连接，那么它们就共享同一个 push cache。
  1. CDN 服务
  • CDN 往往被用来存放静态资源。
  • CDN 的域名必须和主业务服务器的域名不一样，避免不必要的 cookie 的出现。
  1. 图片优化
  • 减少像素点
  • 减少每个像素点能够显示的颜色
  1. 渲染优化

  • 基于渲染流程的 CSS 优化。

    • CSS 选择器是从右到左进行匹配。
      • 避免使用通配符。
      • 避免重复匹配、重复定义。
      • 少用标签选择器。
      • 减少嵌套、最高不超过三层，尽可能使用类关联。

  • CSS 阻塞

    将 CSS 放在 head 标签里和尽快启用CND实现静态资源加载速度的优化。

  • DOM 渲染优化

    • 重绘不一定导致回流，回流一定导致重绘。
    • 尽可能把回流和重绘次数最小化。
  1. Event Loop（事件循环）
  • 事件循环中的异步队列有两种：
    • macro-task（宏任务）：
      • setTimeout、setInterval、setImmediate、script、I/O操作、UI 渲染等。
    • micro-task（微任务）：
      • process、nextTick、promise、mutationObserve等

  1. 节流和防抖

    对有可以被频繁触发的事件进行节流和防抖。
